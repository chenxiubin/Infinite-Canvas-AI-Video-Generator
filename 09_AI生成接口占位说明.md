# 09_AI生成接口占位说明

## 1. 背景与目的

图生视频模型（可灵/Runway/即梦等）与图片生成模型的**具体选型尚未确定**，测试与横向对比计划见《无限画布AI视频生成平台_产品开发需求文档.docx》第九章"待确认事项"。为不阻塞前后端并行开发，本文档定义一套**占位接口契约**，平台内部业务逻辑（04中的封装接口）统一对接此契约，后续替换真实模型 API 时只需实现同一契约，不影响上层业务代码。

## 2. 技术边界重申（对应01产品边界说明）

- 默认路线：**单图 + 约束运镜**，不依赖多角度实拍图的 3D 一致性重建
- 可选路线：涉及产品印刷文字/日期等内容的近景镜头，采用"背景AI生成层 + 原始素材固定图层叠加"方案，文字/数字部分不经过模型重绘（该方案的具体工程实现属于技术选型阶段的独立评估项，本文档仅在接口层面预留字段）

## 3. 占位接口一：图生视频生成（分镜节点使用）

### 请求
`POST /internal/model-gateway/image-to-video`
```json
{
  "source_image_url": "https://.../main.jpg",
  "prompt_fields": {
    "shot_size": "中景",
    "camera_move": "缓慢推进",
    "lighting_mood": "暖光节日氛围",
    "motion_intensity": "轻微"
  },
  "duration_seconds": 4,
  "text_lock": {
    "enabled": false,
    "region_polygon": null
  }
}
```

### 响应（异步任务）
```json
{ "job_id": "model_job_xxx", "status": "queued" }
```

### 轮询结果
`GET /internal/model-gateway/jobs/{job_id}`
```json
{
  "status": "success",
  "output_video_url": "https://.../segment.mp4",
  "actual_duration_seconds": 4.1,
  "model_provider": "placeholder",
  "model_version": "mock-0.1"
}
```

## 4. 占位接口二：AI生成节点候选图生成

### 请求
`POST /internal/model-gateway/text-to-image`
```json
{
  "reference_image_url": "https://.../main.jpg",
  "prompt_fields": {
    "theme": "节日氛围",
    "composition": "居中留白",
    "color_tone": "跟随参考图自动提取"
  },
  "candidate_count": 4
}
```

### 响应
```json
{
  "job_id": "img_job_xxx",
  "status": "success",
  "candidates": [
    { "candidate_id": "c1", "image_url": "https://.../c1.jpg" },
    { "candidate_id": "c2", "image_url": "https://.../c2.jpg" }
  ]
}
```

## 5. Mock 实现建议（MVP-1 阶段）

- 图生视频占位：返回一段固定时长的纯色/占位视频片段（可用 FFmpeg 本地生成），`status` 按固定延时（如 3 秒）从 `queued` → `success`，用于验证前端轮询、状态流转、画布回填逻辑
- 图片生成占位：直接返回参考图本身或裁切变体作为"候选"，用于验证候选选定、锁定缓存逻辑
- Mock 服务应实现与真实模型网关**完全一致的接口契约**（路径、字段、状态枚举），确保替换时前端零改动

## 6. 真实模型接入的预期改动范围

替换为真实模型 API 时，预期仅需改动：
- `model-gateway` 内部适配层（请求参数映射到具体厂商 API 格式）
- 状态轮询频率与超时时间（根据真实模型的生成耗时调整）
- 错误码映射（厂商特定错误 → 平台统一 `error_code`）

不应影响：`04接口草案`中面向前端的业务接口结构、`03数据模型`中的状态机定义。

## 7. 待确认事项（与01/PRD 对应）

- 具体模型厂商选型（可灵/Runway/即梦等）及横向测试计划
- 各厂商单次生成耗时、并发限制、成本，直接影响批量任务的排队/并发策略（对应04的批量接口设计）
- "背景AI生成层+固定图层叠加"方案的具体工程实现方式，需与技术团队评估后再细化本文档的 `text_lock` 字段结构
