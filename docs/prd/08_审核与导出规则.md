# 08_审核与导出规则

## 1. AI生成节点候选审核规则

- 触发条件：仅当素材清单中某角色确实无法拍摄/制作到位时启用（详见07），不作为默认路径
- 单次生成 3～4 个候选，状态进入 `pending_selection`（待选，黄色边框标记，见06）
- 候选生成**不阻塞**批量流程：其他节点/其他产品链正常推进，仅该节点等待人工确认
- 人工选定候选后进入 `locked` 状态并缓存，**后续批量重跑不再重复生成或弹出新候选**
- 若候选均不满意，可修改提示词字段（结构化预设，非自由文本）后重新触发生成，或改为手动上传图片替代，均不阻塞其他角色/其他产品链的进度

## 2. 分镜片段生成结果审核规则

- 分镜节点生成成功后，画布上节点缩略图自动切换为视频首帧，可点击直接预览播放
- 不设置强制人工审核环节（避免拖慢批量效率），但导出前系统会做**基础质量校验**：
  - 校验片段实际时长是否落在该节点设定的 3～5 秒范围内（容差 ±0.2s）
  - 校验片段是否成功生成（无损坏/空白帧），失败则节点状态标记为 `failed`，不进入合成环节
- 人工可在任意时间点开节点预览播放，判断是否需要重跑，重跑规则见下

## 3. 局部重跑规则

- 触发范围：单个分镜节点、单个 AI生成节点候选
- 重跑不影响：同一产品链中其他已成功节点的状态、其他产品链的生成进度
- 重跑次数：MVP 阶段不设上限，由人工判断是否继续重跑或改用其他素材
- 重跑期间该节点状态变更为 `generating`，合成节点在所有节点均为 `success` 前不会自动触发

## 4. 合成与导出触发条件

| 条件 | 规则 |
|---|---|
| 单链合成触发 | 该产品链下所有分镜节点状态均为 `success` 时，合成节点自动触发拼接 |
| 部分失败是否允许导出 | **不允许**导出包含失败节点的成片；若业务上需要"先导出已完成部分产品链、失败的后补"，在批量任务粒度支持，而非单链内部分导出 |
| 批量导出触发 | 批量任务中已 `completed` 的产品链可随时单独导出，无需等待整批 `completed`（对应 `partially_completed` 状态，见03） |

## 5. 导出规格

| 平台预设 | 分辨率/比例 | 适用场景 |
|---|---|---|
| `douyin_9_16` | 9:16 | 抖音/小红书/视频号短视频 |
| `detail_16_9` | 16:9 | 淘宝详情页横版 |
| `detail_1_1` | 1:1 | 淘宝详情页方形 |
| `video_channel` | 9:16 | 视频号（与抖音规格一致，独立配置便于后续差异化） |

- 导出参数（转场、字幕样式、BGM、品牌水印）由合成节点在生成阶段统一处理，导出环节仅做分辨率/比例适配，不重新触发生成任务
- 同一产品链可重复导出多个平台规格，不需要重新生成分镜片段

## 6. 导出文件命名规则

`{产品线}_{产品编号}_{平台预设}_{导出时间戳}.mp4`
例：`hanging_calendar_SKU2027A01_douyin_9_16_20260701.mp4`

统一命名便于批量下载后按产品/平台快速识别归类。

## 7. 版面/文字保护方案的审核触发（可选功能，见09技术边界）

若某分镜节点启用 `text_lock_enabled`（版面/文字保护叠加），生成结果需额外校验：
- 校验叠加后的文字/数字区域是否清晰无形变（MVP 阶段以人工抽查为主，暂不做自动化清晰度检测）
- 该功能技术实现方式待与技术团队评估后细化自动化校验规则（对应01文档待确认事项）
