# 10_开发验收标准

> 验收项按 MVP-1 / MVP-2 / V1 分类，对应《05_MVP开发阶段拆分.md》。格式：Given-When-Then。

## MVP-1 验收项

### 画布基础交互
- [ ] Given 空白画布，When 用户滚轮缩放/拖拽平移，Then 画布响应流畅无明显卡顿（60fps 参考标准）
- [ ] Given 画布上存在节点，When 用户拖拽节点，Then 节点位置更新且不改变连线逻辑顺序

### 节点体系
- [ ] Given 一条分镜模板链，When 节点处于不同生成状态，Then 边框颜色按06规则正确展示（灰/蓝跑马灯/绿/红）
- [ ] Given 用户点击某节点，When 抽屉展开，Then 字段均为下拉预设，无自由文本输入框
- [ ] Given 用户拖动时长滑块，When 数值变化，Then 节点卡片宽度同步变化，且总时长实时刷新

### 素材绑定（MVP-1 手动模式）
- [ ] Given 素材节点为空，When 用户上传图片并选择角色，Then 图片正确绑定到对应角色的分镜节点

### 分镜生成
- [ ] Given 一个分镜节点已绑定素材，When 用户点击生成，Then 请求 09 占位接口，节点状态从 pending → generating → success/failed
- [ ] Given 某节点生成失败，When 用户点击重跑，Then 仅该节点重新生成，其他节点状态不受影响

### 合成与导出
- [ ] Given 一条链所有节点均为 success，When 合成节点触发，Then 自动拼接并应用转场，生成完整视频
- [ ] Given 成片已生成，When 用户选择导出规格并导出，Then 输出符合08命名规则的文件，分辨率/比例正确
- [ ] Given 链条总时长不在 25～30 秒区间，When 用户尝试提交生成，Then 系统阻止提交并提示调整

## MVP-2 验收项

### 批量克隆
- [ ] Given 素材库中框选 N 组产品素材，When 拖拽到已配置模板链上，Then 自动生成 N 条独立产品链实例，且预览动画正确展示裂变过程
- [ ] Given 批量克隆已完成，When 用户撤销（Ctrl+Z），Then N 条实例及关联生成任务同步取消

### 批量任务队列
- [ ] Given 提交了 20 条产品链的批量生成，When 部分链生成失败，Then 批量任务整体状态为 `partially_completed`，不阻塞其他链
- [ ] Given 批量任务看板，When 查看汇总进度，Then 显示格式为"20条链，已完成14，生成中3，失败3"，与后端状态一致
- [ ] Given 用户点击失败项，When 跳转，Then 精确定位并高亮到画布中对应的失败节点

### 双产品线
- [ ] Given 台历模板与挂历模板，When 分别配置差异节点（细节特写2/运镜展示/场景镜头），Then 两条产品线的固定字段可独立配置且互不影响

### 模板复用
- [ ] Given 一条已配置好的分镜链，When 点击"存为模板"，Then 保存内容仅含结构与固定字段，不含任何 asset_id
- [ ] Given 已保存模板，When 在新画布拖入该模板，Then 生成空变量位的节点链，等待绑定新素材

## V1 验收项

### 素材需求清单
- [ ] Given 产品线清单已配置，When 素材未上传，Then 对应节点显示"待补充：{角色名}"标记
- [ ] Given 素材按角色上传完成，When 匹配成功，Then 节点标记由"待补充"变为正常状态
- [ ] Given 命名规范的素材批量上传，When 系统识别，Then 自动匹配角色，识别失败项进入人工标注队列

### AI生成节点
- [ ] Given 某产品某角色素材缺失，When 触发生成候选，Then 3～4秒内（或约定时限内）返回候选列表，节点状态为 `pending_selection`
- [ ] Given 用户选定候选，When 锁定，Then 状态变为 `locked`，后续该产品批量重跑不再重复生成或弹出新候选
- [ ] Given 生成候选中的产品链，When 查看批量任务进度，Then 其他产品链的生成/导出不受该候选待选状态阻塞

### 版面/文字保护（若本阶段实现）
- [ ] Given 节点启用 `text_lock_enabled`，When 生成完成，Then 版面文字区域保持原始素材像素，人工抽查确认无明显模糊/错位

## 通用非功能验收

- [ ] 批量任务在至少 20 条并发规模下，系统响应与状态更新无明显延迟或状态错乱
- [ ] AI能力层的模型调用封装可在不改动上层业务接口的前提下替换供应商（对应09第6节）
- [ ] 所有生成类、导出类接口均为异步设计，前端不存在长时间同步阻塞等待
