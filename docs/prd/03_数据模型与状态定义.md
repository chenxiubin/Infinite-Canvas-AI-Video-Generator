# 03_数据模型与状态定义

## 1. 核心实体总览

```
Product（产品线：台历/挂历）
 └─ Checklist（素材需求清单，属于产品线）
     └─ ChecklistItem（清单项，即素材角色定义）

Template（分镜模板，属于产品线）
 └─ TemplateNode（模板中的节点定义：结构+固定字段，不含图片）
     └─ TemplateEdge（节点间连线：顺序+转场）

Canvas（画布，一个工作空间）
 └─ ProductInstance（一条产品链实例，由模板克隆而来，绑定具体产品）
     └─ InstanceNode（实例节点：AssetNode / GenerationNode / ShotNode / MergeNode）
         └─ AssetBinding（素材绑定关系：角色 → 具体素材文件）

BatchTask（批量任务，包含多个 ProductInstance 的生成请求）
 └─ BatchTaskItem（单条产品链在本次批量中的执行状态）
     └─ NodeGenerationJob（单个分镜节点的生成任务，对应09中的生成接口调用）

ExportJob（导出任务，将某个 ProductInstance 的所有片段合成导出）
```

## 2. 实体字段定义

### 2.1 Product 产品线
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 产品线ID |
| name | string | 台历 / 挂历 |
| default_template_id | string | 默认关联的分镜模板 |

### 2.2 Checklist / ChecklistItem 素材需求清单
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 清单项ID |
| product_id | string | 所属产品线 |
| role_key | string | 角色标识，如 `main` `detail_1` `detail_2` `scene` |
| role_name | string | 展示名称，如"主图-正面" |
| bound_node_key | string | 对应模板节点的标识 |
| content_requirement | string | 内容要求描述 |
| composition_suggestion | string | 建议景别/构图 |
| status | enum | `pending`（待补充）/ `fulfilled`（已到位） |

### 2.3 Template / TemplateNode / TemplateEdge 分镜模板
| 字段 | 类型 | 说明 |
|---|---|---|
| template.id | string | 模板ID |
| template.product_id | string | 所属产品线 |
| template.total_duration_range | [number, number] | [25, 30] 秒 |
| node.node_key | string | 模板内节点唯一标识（如 `S01_main` `S02_detail1`） |
| node.node_type | enum | `asset` / `generation` / `shot` / `merge` |
| node.role_key | string | 关联的素材角色（asset/generation/shot 类型节点适用） |
| node.duration | number | 3～5 秒 |
| node.shot_size | enum | `全景` / `中景` / `特写` |
| node.camera_move | enum | `推` / `拉` / `摇` / `移` / `环绕` / `静止` |
| node.lighting_mood | enum | 预设枚举，产品线维度统一 |
| node.motion_intensity | enum | `轻微` / `中等` / `较大` |
| node.text_lock_enabled | boolean | 是否启用版面/文字保护叠加 |
| node.is_fixed | boolean | 是否为品牌固定镜头（不随产品变化） |
| edge.from_node_key / to_node_key | string | 连接关系 |
| edge.transition_type | enum | `淡入` / `交叉溶解` / `直切` |
| edge.transition_duration | number | 秒 |

### 2.4 ProductInstance / InstanceNode / AssetBinding
| 字段 | 类型 | 说明 |
|---|---|---|
| instance.id | string | 实例ID（一条产品链） |
| instance.template_id | string | 来源模板 |
| instance.product_sku | string | 具体产品编号 |
| instance.total_duration | number | 实际计算总时长 |
| instance_node.status | enum | 见「3.1 节点生成状态机」 |

| asset_binding.source_type | enum | `uploaded`（人工上传）/ `generated`（AI生成节点产出） |
| asset_binding.asset_id | string | 关联素材文件 |

### 2.5 BatchTask / BatchTaskItem / NodeGenerationJob
| 字段 | 类型 | 说明 |
|---|---|---|
| batch_task.id | string | 批次ID |
| batch_task.status | enum | 见「3.2 批量任务状态机」 |
| batch_task_item.instance_id | string | 对应产品链 |
| batch_task_item.status | enum | 同批量任务状态枚举，颗粒度到单链 |
| node_job.instance_node_id | string | 对应实例节点 |
| node_job.status | enum | 见「3.1 节点生成状态机」 |

### 2.6 ExportJob 导出任务
| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 导出任务ID |
| instance_id | string | 关联产品链 |

## 3. 状态机定义

### 3.1 节点生成状态机（InstanceNode / NodeGenerationJob 通用）

```
未生成 pending
  → 生成中 generating
      → 成功 success
      → 失败 failed
          → 重跑 → 生成中 generating（不影响其他节点）
```

对应画布视觉：灰色边框=`pending`，蓝色跑马灯=`generating`，绿色=`success`，红色=`failed`（详见 06）。

### 3.2 批量任务状态机（BatchTask / BatchTaskItem）

```
排队 queued
  → 运行中 running
      → 全部成功 completed
      → 部分完成 partially_completed（存在失败项，但不阻塞其他项）
      → 全部失败 failed
```

`partially_completed` 是批量场景下的常态，不视为异常，允许对失败项单独重跑而不影响已完成项。

### 3.3 AI生成节点候选状态机

```
未触发 not_triggered
  → 生成候选中 candidates_generating
      → 待选 pending_selection（3~4个候选，等待人工选定）
          → 已锁定 locked（选定后缓存，后续批量重跑不再重复生成）
      → 需重新生成 regenerate_requested（候选均不满意，修改提示词字段后重新触发）
```

### 3.4 素材清单角色状态机

```
待补充 pending → 已到位 fulfilled
```

`fulfilled` 触发条件：该角色存在有效的 `AssetBinding`（无论 `source_type` 是 `uploaded` 还是 `generated`）。

## 4. 关键约束

- `ProductInstance.total_duration` 必须在 `Template.total_duration_range` 区间内，超出时前端阻止提交生成任务（见 10 验收标准）
- `TemplateNode` 保存时不包含任何 `asset_id`，克隆生成 `InstanceNode` 时才产生具体绑定
- `NodeGenerationJob` 失败重跑只重置该节点状态，不级联影响同链其他节点或其他 `ProductInstance`
