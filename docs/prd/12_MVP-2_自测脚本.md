# 12_MVP-2_自测脚本

> 用途：MVP-2（批量生产能力）全部 E2E 验收用例的权威定义。用例的代码实现在
> `frontend/tests/mvp2.spec.ts`（G/H/I/J/K 组）和 `backend/tests/test_security.py`
> （安全校验）。

## 测试环境

- 后端：`TESTING=false python -m uvicorn app.main:app --host 127.0.0.1 --port 8000`
- 前端：`npx vite --host 127.0.0.1 --port 5173`
- 运行全量回归：`npx playwright test --grep "G1|G2|H1|H2|H3|I1|I2|I3|I4|J1|J3|J4|K1|K2"`
- 安全测试：`python -m unittest discover -s ../backend/tests`
- 注意：K2 完整隔离性验证需要 `TESTING=true`，否则仅验证 403 安全拦截

## K 组：高级批量场景（失败重跑、精确注入、安全校验）

### K1. 手动修复失败节点全流程端到端验证
- [ ] 批量克隆不完整链（A03 仅有 main），生成后批次"全部失败"
- [ ] 在画布中依次为 detail_1/detail_2/motion/scene 节点绑定素材（前端 UI 操作）
- [ ] 通过 `GET /api/v1/instances/{id}` 验证每个节点的 `boundAssetUrl` 已持久化
- [ ] 逐个点击"生成"按钮，通过 API 轮询确认节点 status → `success`
- [ ] 所有 5 个 shot 节点 success 后，自动触发 merge → instance `completed`
- [ ] 批次状态自愈：`completed_count=1, failed_count=0, status=completed`
- [ ] 批次条目 error_message 更新为"✨ 已手动修复"
- **验证手段**：API 轮询后端状态（`request.get`），不依赖 UI 文本

### K2. force_statuses 精确失败注入——节点级隔离性验证
- **安全校验（TESTING=false）**：
  - [ ] 用合法的 `instance_id:node_key` 复合 key 格式传 force_statuses → 后端返回 403
  - [ ] 批次状态保持 `queued`，无任何节点被强制修改
- **隔离性验证（TESTING=true）**：
  - [ ] 批量克隆 2 条完整链（ins_01、ins_02），通过 `force_statuses` 指定
    `{"ins_01:S03_detail2": "failed"}`
  - [ ] ins_01 的 S03_detail2 节点最终状态为 `failed`
  - [ ] ins_01 的其他 5 个 shot 节点（S01/S02/S04/S05/S06）状态均为 `success`（隔离性）
  - [ ] ins_02 完全不受影响：全部 6 个节点 `success`，instance `completed`，`merged_video_url` 已设置
  - [ ] 批次最终状态 `partially_completed`，`completed_count=1, failed_count=1`
- **代码位置**：`frontend/tests/mvp2.spec.ts` K2

### K3. 批量生成安全校验（后端单元测试）
- [ ] `force_status` 参数在 TESTING=false 时返回 403
- [ ] `force_status` 参数在 TESTING=true 时允许通过（继续到数据库查询）
- [ ] `force_statuses` 在 TESTING=false 时返回 403（即使用的是合法复合 key 格式 + 真实 batch_id）
- [ ] `force_statuses` 在 TESTING=true 时允许通过
- [ ] 新增：用真实 canvas + batch 验证 403 在 force_statuses 路由之前触发，批次未被污染
- **代码位置**：`backend/tests/test_security.py`（5 条测试）
