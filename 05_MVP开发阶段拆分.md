# 05_MVP开发阶段拆分

> 时间为相对估时参考，非最终排期承诺，需结合团队实际人力评估。

## 阶段总览

| 阶段 | 目标 | 预估周期 |
|---|---|---|
| MVP-1 | 单产品线跑通"画布→模板→单条出片"完整链路 | 4～6 周 |
| MVP-2 | 接入批量生产能力 + 第二产品线 | 3～4 周 |
| V1 | 素材清单管理、AI生成节点、模板库独立页 | 4～5 周 |
| V2 | 模板市场化、智能素材推荐、多平台一键分发 | 待评估 |

## MVP-1：单链路验证（建议先做挂历）

**功能范围**
- [ ] 无限画布基础能力：缩放/平移/拖拽/连线（对应06）
- [ ] 四类节点的基础渲染与状态展示（灰/蓝/绿/红状态机，对应03/06）
- [ ] 单条分镜模板的创建与编辑（节点标准字段下拉预设，对应07）
- [ ] 素材节点：单张上传、手动绑定到节点变量位（角色绑定先支持手动，暂不做命名自动识别）
- [ ] 分镜节点生成：对接占位生成接口（09），跑通"提交→轮询→回填片段"
- [ ] 合成节点：自动拼接、加转场，输出成片
- [ ] 导出：至少支持一种平台规格（如 9:16）

**明确不做**
- 不做批量克隆（仅支持单条链手动配置）
- 不做 AI 生成节点的正式接入（用占位/mock 数据验证节点存在即可）
- 不做素材需求清单管理面板

**验收目标**：一条挂历产品链，从空白画布到导出 25～30 秒成片，全流程无阻塞。

## MVP-2：批量能力 + 双产品线

**功能范围**
- [ ] 批量克隆：框选多组素材拖拽到模板链，自动生成 N 条独立实例（对应02流程二）
- [ ] 批量任务队列与看板：汇总进度、单节点失败重跑（对应03状态机）
- [ ] 台历产品线模板接入，验证运镜展示/场景镜头两个差异节点的可配置性（对应07）
- [ ] 模板保存与复用（存为模板，仅保存结构不保存图片）

**验收目标**：20 条产品链的批量任务能够并发提交、独立追踪状态、局部失败不阻塞整体。

## V1：清单管理 + AI生成节点正式接入

**功能范围**
- [ ] 素材需求清单配置与管理页（对应02、07）
- [ ] 清单角色与画布节点的"待补充/已到位"状态联动
- [ ] AI生成节点正式接入图生视频/图片生成模型 API（替换占位接口，对应09）
- [ ] 生成候选审核交互：待选状态标记、批量场景下的快速审核流（对应08）
- [ ] 版面/文字保护叠加方案技术验证（如涉及印刷文字类近景镜头，对应08）
- [ ] 模板库独立页面，支持跨画布管理

**验收目标**：素材缺口场景下，AI生成节点可作为兜底手段补全并入批量流程，不阻塞整体生产节奏。

## V2：规模化与生态（待评估，暂不细化任务拆分）

- 模板市场化（可分享/收藏其他团队沉淀的模板）
- 基于历史转化数据的素材/模板智能推荐
- 多平台一键分发
