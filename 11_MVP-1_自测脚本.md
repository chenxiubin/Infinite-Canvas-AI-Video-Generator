# 11_MVP-1_自测脚本

> 用途：在进入 MVP-2（批量能力开发）前，对当前实现做一轮**手动回归测试**，验证单链路径是否真正稳定。批量能力是在单链基础上做 N 倍复制，单链里的隐藏问题一旦进入批量阶段会被放大，排查成本显著上升，因此本脚本所有用例建议**全部通过**再启动 MVP-2。
>
> 对应文档：`03_数据模型与状态定义.md`（状态机）、`06_UI交互规则.md`（交互规则）、`08_审核与导出规则.md`（导出规则）、`10_开发验收标准.md`（MVP-1验收项）

## 测试环境准备

- [ ] 使用挂历产品线（新年挂历），一套完整素材（主图-正面、细节-纸张质感、细节-装订挂绳等，对应画布左侧素材库）
- [ ] 浏览器开发者工具打开 Network 面板，用于观察接口调用与状态轮询
- [ ] 记录当前代码版本/commit hash，便于问题定位

## 是否可进入 MVP-2 的判定标准

- **A组、B组、C组** 是硬性前置条件，**必须全部通过**——这三组分别对应"链路真的能跑通""批量时长治理不失效""批量失败隔离不失效"，任何一项不通过都会在批量阶段被放大成大问题，不建议带病进入 MVP-2
- **D组、E组、F组** 若有个别不通过，属于可并行修复的问题，不强制阻塞 MVP-2 启动，但需在 MVP-2 开发的第一个迭代内一并修复，避免批量克隆出几十条链后这些细节问题被放大到难以排查
